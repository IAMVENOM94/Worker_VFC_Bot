# Work Time Bot — исправленная версия

Что исправлено:
- убран реальный `.env`, вместо него добавлен `.env.example`
- добавлен `.gitignore`
- разделение на `handlers/common.py`, `handlers/employee.py`, `handlers/admin.py`
- добавлен `handlers/fallback.py`, чтобы `/start` и другие команды не перехватывались раньше времени
- добавлен FSM для сценария `Подсчёт за период`
- добавлена кнопка `Моя смена`
- добавлена кнопка `Кто сейчас работает` для админа
- время хранится в UTC, а при отсутствии tz-базы бот не падает и использует UTC
- в `requirements.txt` добавлен `tzdata`
- при старте смены сохраняется `hourly_rate_snapshot`, чтобы старая статистика не ломалась после изменения ставки
- добавлены индексы и простая миграция старой SQLite-схемы
- добавлен `audit_log` для действий админа `/setrate` и `/setrole`
- добавлен глобальный error handler и нормальное логирование

## Запуск
1. Создай `.env` на основе `.env.example`
2. Установи зависимости:
   ```bash
   pip install -r requirements.txt
   ```
3. Запусти бота:
   ```bash
   python bot.py
   ```


## Docker / bothost
1. Добавь свой `.env` на основе `.env.example`
2. Убедись, что рядом с проектом лежит `Dockerfile`
3. Собери и запусти контейнер:
   ```bash
   docker build -t work-time-bot .
   docker run --env-file .env work-time-bot
   ```

### Что важно
- проект уже написан под **aiogram 3.22.0**
- `executor` здесь **не нужен**
- в контейнер не нужно класть реальный `.env` и рабочую базу `*.db`
